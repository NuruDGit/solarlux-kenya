## 3. Payload CMS Collections

Minimum viable schema. Expand as needed.

### Pages
Flexible page builder for marketing pages if marketing wants to edit without dev involvement.
- `title`, `slug`, `hero` (group), `blocks` (array of content blocks), `seo` (group)

### Products
- `name` (text)
- `slug` (text, unique)
- `category` (relationship в†’ ProductCategories)
- `brand` (relationship в†’ Brands)
- `shortDescription` (textarea)
- `description` (Lexical richText)
- `specifications` (array of { label, value })
- `keyFeatures` (array of text)
- `priceFrom` (number, optional)
- `priceCurrency` (select: KES, USD)
- `priceNote` (text вЂ” "Starting from" / "Contact for pricing")
- `warranty` (text)
- `images` (array в†’ Media)
- `datasheet` (upload в†’ Media, PDF)
- `isFeatured` (checkbox)
- `isInStock` (checkbox)
- `applications` (select multi: residential, commercial, hospitality, agricultural)
- `seo` (group: title, description, ogImage)

### ProductCategories
- `name`, `slug`, `description`, `heroImage`, `icon` (Lucide name)

### Projects (Case Studies)
- `title`, `slug`, `location` (text вЂ” e.g., "Watamu, Kilifi")
- `client` (text, optional вЂ” may be anonymised)
- `sector` (select: residential / commercial / hospitality / agricultural)
- `systemSize` (text вЂ” e.g., "10kVA 48V 8kW")
- `components` (array of text or relationship в†’ Products)
- `summary` (textarea)
- `challenge`, `solution`, `outcome` (Lexical richText)
- `images` (array в†’ Media)
- `completedDate` (date)
- `featured` (checkbox)
- `testimonial` (relationship в†’ Testimonials, optional)

### Testimonials
- `quote` (textarea)
- `authorName`, `authorRole` (text)
- `authorImage` (upload в†’ Media)
- `project` (relationship в†’ Projects, optional)
- `rating` (number 1вЂ“5, optional)
- `featured` (checkbox)

### BlogPosts
- `title`, `slug`, `excerpt`, `content` (Lexical)
- `author` (relationship в†’ TeamMembers)
- `category` (select: guides / news / case-studies / tips)
- `tags` (array of text)
- `heroImage` (upload в†’ Media)
- `publishedDate` (date)
- `readTime` (auto-calc or manual)
- `seo` (group)

### TeamMembers
- `name`, `role`, `bio` (richText)
- `photo` (upload в†’ Media)
- `linkedin` (url, optional)
- `order` (number вЂ” display order)
- `isFounder` (checkbox)

### Brands (partner manufacturers)
- `name`, `logo` (upload в†’ Media), `website` (url)

### Media
- Standard Payload Media collection
- Image sizes: `thumbnail` (300w), `card` (600w), `feature` (1200w), `hero` (1920w)
- Alt text required on every upload

### Globals

**SiteSettings**
- `phonePrimary`, `phoneSecondary`, `email`, `emailAlt`
- `addressLine1`, `addressLine2`, `city`, `country`
- `openingHours` (text)
- `whatsappNumber`
- `socialLinks` (array)
- `yearsInBusiness` (number) вЂ” used in "8+ years" stats

**Header**
- `navItems` (array of { label, href, children? })
- `ctaLabel`, `ctaHref`

**Footer**
- `columns` (array of { title, links })
- `legalText`
- `trustBar` (array of { icon, label })

## 6. First Sprint вЂ” Build Order

1. **Foundation** вЂ” install deps, set up Tailwind, drop in `globals.css` tokens, verify fonts load
2. **Layout shell** вЂ” root layout, header, footer, mobile menu, WhatsApp FAB
3. **Shadcn primitives** вЂ” button, input, card, dialog, sheet, accordion
4. **Motion primitives** вЂ” `FadeIn`, `SlideUp`, `Stagger` components
5. **Homepage** вЂ” hero, value props, featured products, stats bar, CTA
6. **About** вЂ” use PDF content
7. **Services overview + sub-pages**
8. **Solutions pages**
9. **Products** вЂ” category pages + detail pages (static data first, then Payload)
10. **Quote page** вЂ” primary conversion
11. **Projects gallery**
12. **Contact**
13. **Payload CMS** вЂ” wire up collections, migrate static content
14. **Blog / Resources**
15. **SEO, sitemap, robots, structured data**
16. **Performance audit, Lighthouse pass**
17. **Deploy**
